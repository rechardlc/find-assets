# 项目规划

## 1. 项目概述

### 1.1 背景与痛点

在量化投资与日常盯盘中，「寻找大级别趋势拐点（周线）」与「捕捉日线爆发临界点（日线）」是两套经典且高效的独立交易策略。项目首先覆盖 A 股市场，随后扩展到数字货币 USDT 永续合约市场，用同一套 EMA 与形态策略扫描不同资产。

| 痛点 | 说明 |
|------|------|
| 传统软件公式系统并发慢 | 通达信、同花顺等客户端公式无法高效全市场扫描 |
| 多周期联合筛选繁琐 | 周线 + 日线联动需大量手工配置 |
| 无法脱离客户端独立运行 | 难以集成到自动化流程或脚本 |

### 1.2 项目定位

一款面向**个人投资者或量化交易员**的本地化、高并发、多策略独立资产扫描工具。

### 1.3 项目目标

- 利用 Go 语言高并发优势，打造轻量级、零依赖选股可执行工具
- 默认覆盖全 A 股（沪深主板、创业板、科创板），保持现有 A 股入口不变
- 新增独立数字货币入口，覆盖 Binance / OKX USDT 永续热门山寨合约池
- A 股单次全量扫描耗时控制在 **15 秒以内**
- 支持 A 股 CLI 与 HTTP API 双模式，数字货币合约 CLI 独立运行
- 结果可导出 Console / JSON / Markdown

## 2. 核心功能需求

系统把策略拆成两个**正交维度**，通过启动参数或 API 自由组合：

- **周期（`-period` / `-interval`）**：`day`（日线）/ `week`（周线）/ `15m`（15 分钟线），决定指标算在哪种 K 线上；
- **形态（`-pattern`）**：`pierce`（一箭穿心）/ `reversal`（超跌拐点），决定怎样才算命中。

二者交叉得到 4 种策略（`day:pierce`、`week:reversal`、`week:pierce`、`day:reversal`）。
下文描述的「周线超跌拐点」「日线一箭穿心」是其中两组经典组合，其余组合复用同一套形态逻辑。

### 2.1 形态：超跌拐点（`-pattern=reversal`，经典用法 `-period=week`）

**核心目的**：寻找长期下跌后，大级别动能衰竭、均线开始交叉复苏的左侧/右侧过渡标的。

**业务逻辑**：

1. **数据过滤**：拉取全 A 股历史 K 线，剔除退市、ST 及状态异常股票
2. **生命周期兼容**：
   - 老股（上市 ≥ 120 周）：计算 EMA5/10/30/60/120
   - 新股（60 周 ≤ 上市 < 120 周）：仅计算 EMA5/10/30/60
   - 上市不足 60 周：直接淘汰
3. **交叉窗口校验**：
   - 老股：最近 3 周内 EMA60 与 EMA120 发生过金叉或死叉
   - 新股：最近 3 周内 EMA30 与 EMA60 发生过金叉或死叉
4. **形态确认（当根）**：
   - 老股：EMA5 < EMA10 < EMA30 < EMA60 < EMA120（严格空头排列）
   - 新股：EMA5 < EMA10 < EMA30 < EMA60（严格空头排列）

> 上述「周数」门槛为周线（`-period=week`）默认值；用于日线（`-period=day`）时按交易日近似放大
> （新股 ≥ 120 根、老股 ≥ 250 根、交叉窗口 5 根），由形态构造时按周期自动注入。
> 数字货币 `15m` 周期直接使用交易所返回的 15 分钟 K 线，不做日线重采样。

### 2.2 形态：一箭穿心（`-pattern=pierce`，经典用法 `-period=day`）

**核心目的**：寻找均线高度缠绕、筹码极度集中后，主力资金暴力破局的多头启动点。

**业务逻辑**：

1. **数据过滤**：上市天数 ≥ 125 天（按交易日 K 线根数近似）
2. **均线粘合度评估**：
   - 计算 EMA5/10/30/60/120
   - `(MaxEMA - MinEMA) / MinEMA ≤ 2%`（可通过 `-range` 参数自定义，单位为百分比）
3. **K 线形态突破（一箭穿心）**：
   - 当天为阳线（Close > Open）
   - 最高价 > MaxEMA 且 最低价 < MinEMA
4. **成交量放量**：
   - 当天成交量较前一日增加 ≥ 20%（可通过 `-volume` 参数自定义，单位为百分比）

### 2.3 市场：数字货币 USDT 永续合约

**核心目的**：独立运行数字货币扫描器，优先寻找热门山寨合约中的 15 分钟超跌拐点机会。

**业务逻辑**：

1. **市场入口独立**：新增 `cmd/crypto-scanner`，不改变 A 股 `cmd/scanner` 的默认行为
2. **数据源回退**：优先 Binance Futures，失败后回退 OKX Public API
3. **合约范围**：只扫描 USDT 永续合约，排除 BTC、ETH、稳定币及交易状态异常合约
4. **热门池规则**：默认 `hot_alt`，综合 24h 振幅、涨跌幅绝对值、成交额、资金费率异常构造热度分
5. **每日缓存**：每天第一次检索合约池并写入本地缓存，同日后续运行复用缓存
6. **定时扫描**：默认启用 `-schedule`，对齐 15 分钟 K 线收盘并延迟执行，默认 `-delay=20s`；单次扫描可传 `-schedule=false`
7. **策略复用**：默认运行 `15m:reversal`，输出复用 Console / JSON / Markdown exporter
8. **邮件通知**：命中时默认通过 QQ 邮箱 SMTP 推送到 `richard_0525@foxmail.com`，授权码从环境变量读取

## 3. 非功能性需求

| 类别 | 要求 |
|------|------|
| 性能 | 5000+ 股票单次扫描 ≤ 15 秒 |
| 并发 | Goroutine + Channel，最大网络并发 80~100 |
| 数据 | 仅请求 600 根前复权日线，周线本地合成 |
| 交付 | 独立二进制，Windows / Linux 绿色运行 |
| 交互 | CLI 实时进度 + 命中清单；HTTP API + SSE 进度 |

数字货币补充要求：

| 类别 | 要求 |
|------|------|
| 数据源 | Binance 优先，OKX 回退 |
| 合约池 | 每日首次检索并缓存热门山寨 USDT 永续合约 |
| 调度 | 15m K 线边界后延迟执行，默认延迟 20 秒 |
| 隔离 | 独立 `crypto-scanner` 可执行文件，不影响 A 股扫描器 |

## 7. 风险与应对

| 风险 | 应对 |
|------|------|
| 数据源反爬限速 | 并发上限 100、UA 池、指数退避重试 |
| 未复权导致 EMA 失真 | 强制 `fqt=1` 前复权 |
| 当周未收盘周线为伪 K 线 | 文档说明，结果标注日期 |
| 停牌股数据缺口 | 用实际 K 线根数判定生命周期 |
| HTTP 重复触发打爆数据源 | 同时只允许 1 个扫描任务 |
| Binance 网络不可用 | 数字货币扫描器默认回退 OKX |
| 15m 当前 K 线未完全落库 | 对齐边界后延迟执行 |
| 山寨币波动剧烈 | 只输出扫描信号，不接交易账户、不自动下单 |
| 邮箱授权码泄露 | 不写入代码或仓库，优先使用环境变量 `FIND_ASSETS_SMTP_PASS` |
