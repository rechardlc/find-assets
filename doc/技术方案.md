# 技术方案

## 1. 总体架构

采用「**采集 → 合成/归一化 → 计算 → 策略 → 输出**」五层单向数据流。A 股 CLI 与 HTTP 共用 `ScanService` 业务层；数字货币 USDT 永续合约通过独立 `cmd/crypto-scanner` 运行，复用 `indicator`、`strategy` 与 `exporter`。

```
┌─────────────────────────────────────────────┐
│            Entrypoint (cmd/scanner)         │
│   ┌───────────────┐    ┌────────────────┐   │
│   │  CLI 模式     │    │  HTTP 模式      │   │
│   │ -period -pattern│  │ -serve :8080   │   │
│   └──────┬────────┘    └────────┬───────┘   │
│          └─────────┬─────────────┘          │
│                    ▼                        │
│         ┌──────────────────┐                │
│         │   ScanService    │                │
│         └─────────┬────────┘                │
│                   ▼                         │
│   Source → Aggregator → Indicator → Strategy│
│                   ▼                         │
│         ┌──────────────────┐                │
│         │    Exporter      │                │
│         │ Console/JSON/MD  │                │
│         └──────────────────┘                │
└─────────────────────────────────────────────┘

┌─────────────────────────────────────────────┐
│        Entrypoint (cmd/crypto-scanner)      │
│   hot_alt pool + daily cache + 15m schedule │
│                    ▼                        │
│          Binance → OKX CompositeSource      │
│                    ▼                        │
│          Crypto Service → Strategy          │
│                    ▼                        │
│             Exporter Console/JSON/MD        │
└─────────────────────────────────────────────┘
```

数字货币数据流：

```text
1. PreparePoolCache()     → 当日 hot_alt 缓存路径
2. ListAssets()           → []Asset (Binance 优先，OKX 回退)
3. Klines(symbol, 15m)    → []Kline (交易所 15m K 线)
4. Strategy.Get(15m, reversal)
5. Pattern.Eval(bars)     → Result | false
6. Exporter.Write()       → console / json / md
```

## 3. 模块设计

### 3.1 数据源（Source）

**接口**：

```go
type Source interface {
    ListAll(ctx context.Context) ([]model.Stock, error)
    DailyKlines(ctx context.Context, stock model.Stock, n int) ([]model.Kline, error)
}
```

**实现**：东方财富 push2 接口

| 用途 | URL | 关键参数 |
|------|-----|----------|
| 股票清单 | `80.push2.eastmoney.com/api/qt/clist/get` | `fs` 沪深 A 股全集 |
| K 线历史 | `push2his.eastmoney.com/api/qt/stock/kline/get` | `klt=101` 日线，`fqt=1` 前复权，`lmt=600` |

**过滤规则**（`filter.go`）：

- 保留：60/000/001/002/003（主板）、300/301（创业板）、688/689（科创板）
- 剔除：ST/*ST/退市、B 股、北交所、ETF/指数

**HTTP 客户端优化**：

- `MaxIdleConnsPerHost: 300`（默认 2 是性能瓶颈）
- 超时 8s，最多重试 2 次，指数退避 + jitter
- User-Agent 池轮换

### 3.2 K 线合成（Aggregator）

**设计原则**：只向服务器请求 600 根前复权日线，周线在本地合成，一次下载、多策略复用。

**算法**：按 ISO 周编号分组（周一为一周起点）

| 字段 | 规则 |
|------|------|
| Open | 该周第一个交易日开盘价 |
| Close | 该周最后一个交易日收盘价 |
| High | 该周最高价 |
| Low | 该周最低价 |
| Volume | 该周成交量累加 |

当周未收盘时，最后一根周线为「伪 K 线」，符合策略「含当周」语义。

数字货币 `15m` 周期由交易所直接返回目标周期 K 线，`Minute15Period` 做恒等处理，不经过日线重采样。

### 3.3 技术指标（Indicator）

**EMA 递推公式**：

```
alpha = 2 / (N + 1)
EMA[i] = alpha * Close[i] + (1 - alpha) * EMA[i-1]
```

冷启动：`EMA[0] = Close[0]`。

**交叉判定**（`Cross`）：在 `[from, to]` 闭区间内，若 `a[i-1] - b[i-1]` 与 `a[i] - b[i]` 符号相反，则判定发生过金叉或死叉。

### 3.4 选股策略（Strategy = 周期 × 形态）

策略被拆成两个**正交维度**，二者可自由组合：

- **Period（周期）**：决定指标算在哪种 K 线上，只负责把原始日线重采样成目标周期。
- **Pattern（形态）**：面对一段**已经是目标周期**的 K 线做命中判定，不关心日线还是周线。

```go
type Period interface {
    Name() string                               // "15m" / "day" / "week"
    Label() string                              // "15分钟" / "日线" / "周线"
    Resample(daily []model.Kline) []model.Kline // 15m/day 恒等，week 调 ToWeekly
}

type Pattern interface {
    Name() string  // "pierce" / "reversal"
    Label() string // "一箭穿心" / "超跌拐点"
    Eval(stock model.Stock, bars []model.Kline) (model.Result, bool)
}

// Strategy = Period × Pattern
type Strategy interface {
    Period() string  // "15m" / "day" / "week"
    Pattern() string // "pierce" / "reversal"
    Mode() string    // 组合标识 "day:pierce"
    Title() string   // "日线一箭穿心"
    Match(stock model.Stock, daily []model.Kline) (model.Result, bool)
}
```

`Match` 内部固定为「先 `Period.Resample(daily)`，再 `Pattern.Eval(bars)`」。

**策略构造**：`strategy.Get(period, pattern, opt)`，例如 `Get("day", "pierce", Options{Range: 1.2})`。
A 股四种组合：`day:pierce`、`week:reversal`、`week:pierce`、`day:reversal`。数字货币当前主用 `15m:reversal`。

> **阈值随周期而变**：`pierce` 的 `MinBars`、`reversal` 的 `MinBarsNew/OldBars/CrossWindow`
> 这类「生命周期 / 最小根数」参数在不同周期下语义不同，由 `newPierce(period)` / `newReversal(period)`
> 按周期注入默认值，而非写死在形态里；`Range`、`Volume` 可经 `Options` 覆盖。

**命中结果结构**：

```go
type Result struct {
    Code, Name, Tag, Metric string
    Snapshot Snapshot  // 含 EMA 快照、粘合度、参与计算的 K 线根数(Bars) 等
}
```

### 3.5 并发扫描器（Scanner）

采用 **信号量 Channel + WaitGroup + 原子计数** 模式：

```
for each stock:
    sem <- struct{}{}          // 获取并发槽
    go func() {
        klines := src.DailyKlines(stock, 600)
        if strat.Match(stock, klines) { out <- result }
        atomic.AddInt64(&done, 1)
        progress(done, total)
    }()
```

- 默认并发：100
- 结果按股票代码排序
- 支持 `ProgressFn` 回调，供 CLI 打印与 SSE 推送

### 3.6 业务编排（ScanService）

统一编排：拉清单 → 并发扫描 → 组装 `exporter.Report`。

CLI 与 HTTP 均调用 `ScanService.Run()`，避免重复实现。

### 3.7 数字货币扫描（Crypto）

数字货币扫描器独立于 A 股 `source.Source`，使用 `internal/crypto.Source`：

```go
type Source interface {
    Name() string
    ListAssets(ctx context.Context) ([]Asset, error)
    Klines(ctx context.Context, asset Asset, interval string, limit int) ([]model.Kline, error)
}
```

数据源：

| 数据源 | 用途 | 接口 |
|--------|------|------|
| Binance | USDT 永续合约列表、24h ticker、资金费率、K 线 | `/fapi/v1/exchangeInfo`、`/fapi/v1/ticker/24hr`、`/fapi/v1/premiumIndex`、`/fapi/v1/klines` |
| OKX | USDT 永续 ticker、K 线 | `/api/v5/market/tickers?instType=SWAP`、`/api/v5/market/candles` |

`CompositeSource` 默认按 `binance,okx` 顺序回退。合约池通过 `hot_alt` 评分生成：

```text
hot_score = amplitude + abs(price_change_percent) + log10(quote_volume) + abs(funding_rate) * 10000
```

过滤规则：

- 只保留 USDT 永续合约
- 只保留交易状态正常的合约
- 排除 BTC、ETH
- 排除 USDC、FDUSD、TUSD、USDP、DAI 等稳定币

每日合约池缓存位于可执行文件旁：

```text
crypto/pools/hot_alt_YYYYMMDD_<exchange>.json
```

数字货币入口默认开启 `-schedule`，使用 `NextDelayedRun(now, 15m, delay)` 对齐下一根 15m K 线边界，并在收盘后延迟执行，默认 `20s`。如需单次扫描，可传 `-schedule=false`。

命中结果后可通过 `internal/notify` 发送邮件。数字货币入口默认启用邮件通知，QQ SMTP 默认配置为 `smtp.qq.com:465`，收件人、发件人和 SMTP 用户默认均为 `rechard.liu@qq.com`。授权码从 `FIND_ASSETS_SMTP_PASS` 或 `-smtp-pass` 读取；未配置授权码时跳过邮件发送，不影响扫描。

### 3.8 结果导出（Exporter）

| 格式 | 说明 |
|------|------|
| `console` | 控制台文本，符合 PRD 输出规范 |
| `json` | 结构化报告，含 task_id、耗时、Snapshot |
| `md` | Markdown 表格，含 EMA 明细 |

**Report 结构**：

```go
type Report struct {
    TaskID, Period, Pattern, Mode, Title, Elapsed string // Mode = "period:pattern"
    StartedAt, FinishedAt time.Time
    Total, Matched int
    Results []model.Result
}
```

### 3.9 HTTP 服务（Gin）

- 异步任务：`TaskStore` 内存存储，`task_id` 查询
- SSE：`/api/v1/scans/:id/stream` 实时推送进度
- 限流：同时只允许 1 个扫描任务（`atomic.Int32`）
- 优雅关闭：`Shutdown` + 5s 超时

## 4. 数据流

```
1. ListAll()              → []Stock (约 5300 只)
2. DailyKlines(code)      → []Kline (600 根前复权日线)
3. Period.Resample(daily) → []Kline (day 恒等 / week 合成约 120 根周线)
4. EMA(closes, N)         → []float64
5. Pattern.Eval(bars)     → Result | false
6. Exporter.Write()       → console / json / md
```

## 5. 性能设计

| 项目 | 数值 |
|------|------|
| 股票总数 | ~5300 |
| 单次 HTTP 平均耗时 | 150~300 ms |
| 并发数 | 100 |
| 理论耗时 | 5300 / 100 × 0.25 ≈ 13 s |
| CPU（EMA 计算） | < 1 s，可忽略 |

**瓶颈在网络**，优化重点：

1. `MaxIdleConnsPerHost` 放大至 300
2. Keep-Alive 复用连接
3. 可选：按交易日文件缓存（v1.1）

## 6. 前复权与数据校验

| 问题 | 处理 |
|------|------|
| 除权日 EMA 失真 | 强制 `fqt=1` 前复权 |
| 前复权后出现负价 | `Close <= 0` 跳过该 K 线 |
| 停牌占位符 `"-"` | `parseFloat` 返回 0 并过滤 |
| 次新股 K 线不足 | `len(klines) < 60/125` 直接淘汰 |

## 8. 扩展点

| 扩展点 | 方式 |
|--------|------|
| 新数据源 | 实现 `source.Source` 接口 |
| 新周期（如月线/30 分钟） | 实现 `strategy.Period` 并注册到 `resolvePeriod()` / `Periods()` |
| 新形态 | 实现 `strategy.Pattern` 并注册到 `resolvePattern()` / `Patterns()` |
| 新导出格式 | 实现 `exporter.Exporter` 并注册到 `Registry()` |
| 配置外置 | 扩展 `service.Params` 或引入 config 包 |
| 数字货币新交易所 | 实现 `crypto.Source` 接口并加入 `CompositeSource` |
| 数字货币新合约池 | 在 `internal/crypto/pool.go` 增加评分/过滤规则 |
| 新通知渠道 | 在 `internal/notify` 增加实现，并在入口按命中结果触发 |
