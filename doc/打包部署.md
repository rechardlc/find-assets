# 打包部署

## 1. 环境要求

| 项目 | 要求 |
|------|------|
| 编译环境 | Go 1.25+ |
| 运行环境 | **零依赖**，无需 Go / Python / 数据库 |
| 网络 | A 股需能访问东方财富 push2 接口；数字货币需能访问 Binance Futures / OKX Public API |

---

## 2. 本地打包（Windows）

在项目根目录执行：

```powershell
cd c:\Users\33002\Desktop\github\find-assets

# 推荐：压缩体积
go build -ldflags="-s -w" -o find-assets.exe ./cmd/scanner

# 数字货币 USDT 永续合约扫描器
go build -ldflags="-s -w" -o crypto-scanner.exe ./cmd/crypto-scanner
```

参数说明：

| 参数 | 含义 |
|------|------|
| `-ldflags="-s -w"` | 去掉符号表和调试信息，体积约 10~15 MB |
| `-o find-assets.exe` | A 股扫描器输出文件名 |
| `-o crypto-scanner.exe` | 数字货币扫描器输出文件名 |
| `./cmd/scanner` | A 股主程序包路径 |
| `./cmd/crypto-scanner` | 数字货币主程序包路径 |

---

## 3. 交叉编译

### Linux amd64

```powershell
$env:CGO_ENABLED="0"
$env:GOOS="linux"
$env:GOARCH="amd64"
go build -ldflags="-s -w" -o dist/scanner ./cmd/scanner
go build -ldflags="-s -w" -o dist/crypto-scanner ./cmd/crypto-scanner
Remove-Item Env:GOOS, Env:GOARCH, Env:CGO_ENABLED
```

Linux 运行：

```bash
chmod +x scanner
./scanner -period=day -pattern=pierce
./crypto-scanner -source binance,okx -interval 15m -pattern reversal
```

### macOS arm64

```powershell
$env:CGO_ENABLED="0"
$env:GOOS="darwin"
$env:GOARCH="arm64"
go build -ldflags="-s -w" -o dist/scanner-darwin-arm64 ./cmd/scanner
go build -ldflags="-s -w" -o dist/crypto-scanner-darwin-arm64 ./cmd/crypto-scanner
```

## 4. 一键打包脚本（可选）

创建 `build.ps1`：

```powershell
$ErrorActionPreference = "Stop"
New-Item -ItemType Directory -Force -Path dist | Out-Null
$ldflags = "-s -w"

Write-Host "Building Windows..."
go build -ldflags=$ldflags -o dist/find-assets.exe ./cmd/scanner
go build -ldflags=$ldflags -o dist/crypto-scanner.exe ./cmd/crypto-scanner

Write-Host "Building Linux..."
$env:CGO_ENABLED = "0"
$env:GOOS = "linux"
$env:GOARCH = "amd64"
go build -ldflags=$ldflags -o dist/scanner ./cmd/scanner
go build -ldflags=$ldflags -o dist/crypto-scanner ./cmd/crypto-scanner
Remove-Item Env:GOOS, Env:GOARCH, Env:CGO_ENABLED

Write-Host "Done."
Get-ChildItem dist
```

## 5. 运行方式

### CLI 模式

```powershell
# 日线一箭穿心
.\find-assets.exe -period=day -pattern=pierce

# 周线超跌拐点 + 导出
.\find-assets.exe -period=week -pattern=reversal -export=json,md -out=./output

# 自定义参数（粘合度 1.2%，并发 80）
.\find-assets.exe -period=day -pattern=pierce -workers=80 -range=1.2 -bars=600
```

### HTTP 服务模式

```powershell
.\find-assets.exe -serve -addr=:8080
```

访问 `http://localhost:8080/api/v1/health` 验证。

### 数字货币合约 CLI 模式

如需命中时邮件推送，先复制 `env.example` 为 `.env` 并填入 QQ 邮箱 SMTP 授权码：

```powershell
Copy-Item env.example .env
notepad .env
```

`.env` 内容：

```text
FIND_ASSETS_SMTP_PASS=你的QQ邮箱授权码
```

程序默认读取当前目录的 `.env`，也可以通过 `-env` 指定其它路径。

```powershell
# 默认定时扫描：Binance 优先，失败回退 OKX；命中时推送到 rechard.liu@qq.com
.\crypto-scanner.exe

# 单次扫描
.\crypto-scanner.exe -schedule=false

# 关闭邮件通知
.\crypto-scanner.exe -mail=false

# 导出 JSON + Markdown
.\crypto-scanner.exe -export=json,md -out=./output
```

数字货币合约池缓存会写入可执行文件旁：

```text
crypto/pools/hot_alt_YYYYMMDD_<exchange>.json
```

## 8. 常见问题

| 问题 | 处理 |
|------|------|
| 双击 exe 无反应 | CLI 程序需在命令行传参，如 `-period=day -pattern=pierce` |
| 中文乱码 | PowerShell 执行 `chcp 65001`，或使用 Windows Terminal |
| 体积偏大 | Gin 依赖正常，约 10~15 MB；可用 UPX 进一步压缩 |
| 扫描超时 / 失败 | 检查网络能否访问 eastmoney.com |
| 端口占用 | 换 `-addr=:9090` 或其他端口 |
| 数字货币无结果 | 检查是否能访问 Binance / OKX；同日缓存可删除 `crypto/pools` 后重建 |
| 15m 信号延迟 | `-delay` 用于等待交易所 K 线落库，可按网络情况调成 `30s` 或 `60s` |
| 邮件未发送 | 仅命中时发送；确认已开启 QQ 邮箱 SMTP 并设置 `FIND_ASSETS_SMTP_PASS` |

## 9. 可选：UPX 压缩

```powershell
upx --best dist/find-assets.exe
```

可将体积再压缩约 50%，部分杀毒软件可能误报，需自行权衡。
